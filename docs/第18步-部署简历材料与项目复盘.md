## Docker 运行方式

构建镜像：

```bash
docker build -t agentops-studio .
```

运行 MockProvider 版本：

```bash
docker run --rm -p 8000:8000 agentops-studio
```

运行 DeepSeek 版本：

```bash
docker run --rm -p 8000:8000 --env-file .env agentops-studio
```

## 最终验收命令

运行测试：

```bash
python -m pytest
```

运行评测：

```bash
python scripts/run_evals.py
```

启动服务：

```bash
python -m uvicorn app.api.routes:app --reload
```

访问前端：

```text
http://localhost:8000/
```

访问接口文档：

```text
http://localhost:8000/docs
```

## 当前项目模块

| 模块 | 说明 |
|---|---|
| FastAPI API | 提供健康检查、任务运行、文档问答、会话记忆、评测报告和前端入口。 |
| Schema | 使用 Pydantic 约束核心数据结构。 |
| Provider | 支持 MockProvider 和 DeepSeekProvider。 |
| Planner | 把任务输入转成结构化执行计划。 |
| Agent Runtime | 串联计划、执行、反思和最终答案。 |
| Tool Registry | 统一管理 mock 工具和 Markdown 工具。 |
| Markdown QA | 读取本地 Markdown 并返回带证据答案。 |
| Memory | 用内存事件保存会话上下文。 |
| Evaluation | 固定评测集和自动 Markdown 报告。 |
| Web Demo | 浏览器页面体验核心 API。 |

## 项目解决的问题

AgentOps Studio 解决的是 Agent 工程化学习问题：

- 如何把大模型输出变成可校验 schema。
- 如何把不同模型 provider 隔离在统一接口后面。
- 如何处理真实模型失败、重试和 fallback。
- 如何把工具调用做成可测试、可观测的层。
- 如何为文档问答、记忆和评测建立最小可运行链路。
- 如何把后端能力交付成 API、前端 Demo 和部署文档。
