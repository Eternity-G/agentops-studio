# 简历项目描述

## 项目名称

Codebase AgentOps Studio：面向企业研发团队的代码仓库问答与代码理解 Agent

## 一句话描述

构建一个面向企业研发团队的代码仓库理解 Agent，支持仓库结构扫描、代码搜索、Python AST 符号解析、代码问答、影响分析、Diff Review、Trace、评测和前端展示，帮助新人快速熟悉项目、定位功能实现、辅助代码评审和变更风险分析。

## 简历写法

可以写成：

```text
Codebase AgentOps Studio | Python / FastAPI / Pydantic / DeepSeek API / AST / pytest / Docker

- 面向企业研发团队构建代码仓库问答与代码理解 Agent，支持仓库扫描、代码搜索、符号解析、代码问答、影响分析和 Diff Review，服务新人 onboarding、功能定位和代码评审场景。
- 设计安全代码读取与搜索能力，限制仓库根目录、文件类型和输出长度，默认跳过 .env、.git、node_modules 等敏感或高成本目录。
- 基于 Python AST 实现类、函数、方法和 import 符号解析，使系统不仅能做关键词搜索，还能结构化理解 Python 模块。
- 构建 CodebaseQuestionAnswerer，将仓库扫描、关键词搜索、代码片段读取和证据聚合串成代码问答链路，答案返回文件路径、行号和证据片段。
- 实现文件级影响分析能力，通过模块路径、文件名和引用搜索判断变更可能影响的模块，并给出测试建议。
- 实现 Diff Review 能力，读取当前 git diff，识别变更文件、风险点和建议执行的回归测试。
- 保留 ToolRegistry 工具层抽象，将代码搜索、文件读取、Markdown 读取等能力封装为可注册、可测试、可替换的工具边界。
- 抽象 ModelProvider 接口，支持 MockProvider 与 DeepSeekProvider，并为真实模型输出增加重试、schema 再校验和 fallback 机制。
- 建立 EvaluationRunner 和固定评测集，覆盖任务主链路、文档问答、会话记忆和代码仓库问答，并生成 Markdown 评测报告。
- 使用 FastAPI 暴露代码仓库分析 API，并提供浏览器端代码仓库分析控制台和 Docker 部署材料。
```

## 面试讲解主线

建议按这条线讲：

1. 企业为什么需要代码仓库问答：新人上手慢、老项目难改、PR Review 成本高、线上问题排查慢。
2. 为什么不能只做普通聊天机器人：真实研发问题必须返回文件路径、行号和证据。
3. 如何做安全边界：只允许访问仓库根目录内文件，限制文件类型，拒绝 `.env`。
4. 如何做代码理解：仓库扫描、关键词搜索、AST 符号解析、证据聚合。
5. 如何回答问题：基于搜索命中的代码片段，而不是凭空生成。
6. 如何做影响分析：搜索目标文件、模块路径和文件名引用，给出影响区域和测试建议。
7. 如何做 PR Review：分析 git diff，识别 schema、API、前端、依赖配置等风险。
8. 如何保障质量：pytest + EvaluationRunner + 固定评测集 + Markdown 报告。
9. 如何接入真实模型：DeepSeekProvider 负责总结和规划，测试默认使用 MockProvider 保持稳定。

## 项目价值

企业研发团队可以用它做：

- 新人 onboarding：快速理解仓库目录、入口、核心模块。
- 功能定位：查找某个 API、类、函数或业务逻辑在哪里实现。
- 影响分析：判断修改某个文件可能影响哪些模块。
- PR Review：辅助识别改动风险和测试建议。
- 知识沉淀：把代码问答结果写入会话记忆，形成上下文记录。

## 当前边界

这个项目已经具备企业内部工具雏形，但还不是生产级系统。

后续可以继续扩展：

- 向量数据库和 embedding 级代码检索。
- Tree-sitter 多语言 AST 解析。
- GitHub App / GitLab Webhook 集成。
- PR 自动评论。
- 权限控制和多租户隔离。
- 真实模型生成代码修改建议，但仍需人工审批。
