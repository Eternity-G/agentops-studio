# 项目复盘

## 项目定位

Codebase AgentOps Studio 是一个面向企业研发团队的代码仓库问答与代码理解 Agent。

它解决的问题不是“如何调用一次大模型”，而是“如何让研发人员更快理解真实代码仓库，并基于代码证据回答问题、分析影响范围和辅助 PR Review”。

## 实际业务背景

企业研发团队经常面临这些问题：

- 新人加入项目后，需要大量时间理解目录结构、启动方式、核心模块和调用链路。
- 老项目文档过期，开发者只能从代码中寻找真实业务逻辑。
- PR Review 时，Reviewer 需要快速判断改动影响范围、风险点和测试是否充分。
- 线上问题排查时，需要快速定位日志、接口、配置和相关代码路径。

因此，代码仓库问答 Agent 的价值在于：

```text
把复杂代码仓库变成可以提问、可以定位、可以追溯证据的研发助手。
```

## 关键工程决策

### 1. 代码问答必须带证据

代码仓库问答不能只输出自然语言解释。

真实研发场景里，答案必须包含：

- 文件路径。
- 行号。
- 代码片段。
- 风险提示。

否则 Reviewer 或开发者无法确认答案是否可信。

### 2. 安全读取优先于能力扩展

系统默认拒绝读取 `.env`，跳过 `.git`、`node_modules`、构建目录和缓存目录。

这是企业内部工具的基本要求，因为代码仓库中经常包含密钥、缓存和大体积依赖。

### 3. 先做静态分析，再做代码生成

当前系统不自动修改代码。

它先解决企业更常见、更安全的问题：

- 代码理解。
- 功能定位。
- 影响分析。
- PR Review 建议。

生成代码可以作为后续增强，但必须加入人工审批和测试验证。

### 4. 评测独立于真实模型

离线评测默认使用 MockProvider，保证稳定、低成本、可复现。

真实 DeepSeekProvider 可以用于代码解释和总结，但不应该让基础测试依赖网络和 API key。

## 当前限制

- 代码检索仍是关键词搜索，没有 embedding 和向量数据库。
- AST 解析目前重点支持 Python，多语言支持还可以用 Tree-sitter 扩展。
- 影响分析是静态启发式，不等同于完整调用图。
- 会话记忆仍是进程内存储。
- 没有 GitHub App / GitLab Webhook 集成。
- 不自动写代码，只给分析和建议。

## 后续增强方向

1. 引入 Tree-sitter，支持 TypeScript、Go、Java 等多语言符号解析。
2. 引入向量数据库，实现语义级代码检索。
3. 接入 GitHub PR，自动生成 Review 评论。
4. 接 SQLite/PostgreSQL，持久化会话、trace 和评测结果。
5. 引入权限控制，限制不同用户可访问的仓库。
6. 增加真实模型总结层，生成更自然的代码解释和风险分析。
